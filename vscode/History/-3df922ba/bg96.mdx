# Domain DNS Transfer: Hostinger to Cloudflare (Next.js Friendly)

This guide will help you move your domain's DNS from Hostinger to Cloudflare, configure optimal SSL/TLS and performance settings, and ensure compatibility with **Next.js (SSG/SSR)** applications.

---

## 1. Create a Free Cloudflare Account

- Visit: [https://dash.cloudflare.com](https://dash.cloudflare.com)
- Sign up using your Gmail or email account

---

## 2. Add Your Site to Cloudflare

- Click **"+ Add a Site"**
- Enter your domain (e.g. `naturalsefaa.com`)
- Choose the **Free Plan**

---

## 3. Import or Configure DNS Records

Cloudflare will attempt to auto-import records from Hostinger. If not, add manually:

| Type | Name | Value    |
| ---- | ---- | -------- |
| A    | @    | VPS IPv4 |
| A    | www  | VPS IPv4 |
| AAAA | @    | VPS IPv6 |
| AAAA | www  | VPS IPv6 |

> Ensure the **Proxy status** (orange cloud) is ON to enable Cloudflare protection.

---

## 4. Update Hostinger Nameservers

Cloudflare will provide nameservers like:

```text
josh.ns.cloudflare.com
lola.ns.cloudflare.com
```

1. Login to Hostinger
2. Go to **Domains > DNS / Nameservers**
3. Select **Custom Nameservers**
4. Paste Cloudflare's nameservers
5. Save & wait for propagation (can take 15 mins – 12 hours)

Check propagation: [https://dnschecker.org](https://dnschecker.org)

---

## 5. Configure SSL on Cloudflare

Go to **SSL/TLS > Overview**:

- **SSL Mode**: `Full (strict)`
- **Always Use HTTPS**: Enabled
- **Automatic HTTPS Rewrites**: Enabled
- **TLS 1.3**: Enabled
- **Minimum TLS Version**: 1.2

> Ensure your VPS has **Let's Encrypt SSL** enabled if using `Full (strict)`.

---

## 6. Performance Optimization

Navigate to **Speed > Optimization**:

| Option                         | Status   |
| ------------------------------ | -------- |
| Auto Minify (HTML/CSS/JS)      | On       |
| Brotli Compression             | On       |
| Rocket Loader (JS Boost)       | Optional |
| Early Hints                    | Enable   |
| Enhanced HTTP/2 Prioritization | On       |

---

## 7. Caching (Next.js Compatible)

Go to **Caching > Configuration**:

- **Browser Cache TTL**: `1 hour` or `4 hours`
- **Always Online**: Enabled
- **DO NOT enable "Cache Everything"** if using SSR/ISR

> Next.js SSR/ISR depends on dynamic rendering — caching full HTML may break behavior.

---

## 8. Security Settings

Cloudflare → Security > Settings:

- **DDoS Protection**: Enabled
- **Bot Fight Mode**: Enabled
- **IP Access Rules**: Optional geo-blocking

---

## 9. Verify Setup

Use terminal:

```bash
dig naturalsefaa.com
```

Output should show a **Cloudflare IP** like `104.21.x.x`, proving Cloudflare is active.

---

## Notes for Next.js Projects

- Compatible with **App Router**, **SSG**, **SSR**, and **API routes**
- For `Full (strict)` SSL: ensure SSL cert is installed on server (e.g. Let's Encrypt)
- Cloudflare handles edge cache, DDoS, TLS handshake — you manage backend logic

> Need help with NGINX reverse proxy + Cloudflare? Just ask!

---

## Done!

Your domain is now:

- Fast with CDN + Compression
- Secured with HTTPS
- Protected by DDoS + Bot Filtering
- Served globally with lower latency

**Cloudflare + Next.js = Power combo for modern web.**
